# 笔记功能实现计划（增强版）

## 功能概述

在用户存储目录下添加笔记功能，每个笔记分为元数据和内容两个JSON文件存储。

## 存储结构

* 笔记目录：`/storage/{user_uuid}/.note/`

* 元数据文件：`{uuid}_meta.json`

* 内容文件：`{uuid}_content.json`

* 索引文件：`index.json` - 记录所有笔记的元信息，便于快速查询

## 数据结构

### 元数据 (meta)

```json
{
  "uuid": "笔记UUID",
  "title": "笔记标题（可选，默认'无标题'）",
  "created_at": "创建时间ISO格式",
  "updated_at": "更新时间ISO格式"
}
```

### 内容 (content)

```json
{
  "uuid": "笔记UUID",
  "content": "笔记内容"
}
```

### 索引 (index)

```json
{
  "notes": [
    {"uuid": "uuid1", "title": "标题", "updated_at": "时间"},
    {"uuid": "uuid2", "title": "标题", "updated_at": "时间"}
  ]
}
```

## 实现步骤

### 1. 创建笔记模块目录和初始化文件

* 创建 `api/note/` 目录

* 创建 `api/note/__init__.py`

### 2. 创建笔记路由模块

* 创建 `api/note/note.py`，实现以下路由：

  * `/note/add` POST - 添加笔记（参数：uuid，可选：title）

  * `/note/update` POST - 更新笔记（参数：uuid, content，可选：title）

  * `/note/delete` POST - 删除笔记（参数：uuid）

  * `/note/query` POST - 查询笔记（参数：num，返回最新num篇）

### 3. 修改认证中间件

* 在 `auth.py` 中添加 `/note` 路径的鉴权

### 4. 注册路由

* 在 `main.py` 中导入并注册笔记路由

## 优化点

1. **使用索引文件加速查询** - 维护一个index.json记录所有笔记的元信息
2. **元数据和内容合并返回** - query时自动合并两个JSON的数据
3. **自动创建目录** - 确保.note目录存在
4. **删除时同时清理** - 删除笔记时删除两个文件
5. **返回格式统一** - 返回的数据包含元数据和内容，方便前端使用
